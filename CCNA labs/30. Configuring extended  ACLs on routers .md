## Goal

1.  Configure extended ACLSs to fulfill the following network policies:

     a-Hosts in 172.16.2.0/24 can't communicate with PC1.
     b-Hosts in 172.16.1.0/24 can't access the DNS service on SRV1.
     c-Hosts in 172.16.2.0/24 can't access the HTTP or HTTPS services on SRV2.

##  Configuration - ACL (R1)


```
R1(config)ip access-list extened SRV1
R1(config-std-nacl)deny udp 172.16.1.0 0.0.0.255 host 192.168.1.100 eq 53
R1(config-std-nacl)permit ip any any
R1(config)interface g0/0
R1(config-if)ip access-group SRV1 in

# fullfill b 

R1(config)ip access-list extened SRV2
R1(config-std-nacl)deny ip 172.16.2.0 0.0.0.255 host 172.16.1.1
R1(config-std-nacl)deny tcp 172.16.2.0 0.0.0.255 host 192.168.2.100 eq 80
R1(config-std-nacl)deny tcp 172.16.2.0 0.0.0.255 host 192.168.2.100 eq 443
R1(config-std-nacl)permit ip any any
R1(config)interface g0/1
R1(config-if)ip access-group SRV2 in
